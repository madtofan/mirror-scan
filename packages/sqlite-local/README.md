# @mirror-scan/sqlite-local

Local SQLite database for offline transaction ledger in the native app.

## Quick Start

### 1. Generate Migrations

After modifying the schema, generate migration SQL:

```bash
cd packages/sqlite-local
pnpm drizzle-kit generate
```

This creates SQL migration files in `src/migrations/`.

### 2. Use in Native App

Wrap your app with `SQLiteProvider` in `apps/native/app/_layout.tsx`:

```tsx
import { SQLiteProvider } from "expo-sqlite";
import { migrateDbIfNeeded } from "@/lib/ledger";

export default function Layout() {
  return (
    <SQLiteProvider databaseName="sqlite-local.db" onInit={migrateDbIfNeeded}>
      {/* Your app content */}
    </SQLiteProvider>
  );
}
```

### 3. Query the Database

Import and use the ledger functions from `@/lib/ledger`:

```tsx
import { getLatestEntry, getBalance, addTransaction } from "@/lib/ledger";

// Get latest transaction
const entry = await getLatestEntry();

// Get balance for a public key
const balance = await getBalance(userPubKey);

// Add new transaction
const newTx = await addTransaction({
  id: "uuid",
  fromPubKey: "sender_pubkey",
  toPubKey: "recipient_pubkey",
  amount: 1500,
  sequenceNumber: 1,
  signature: "cryptographic_signature",
});
```

## Schema

The ledger table stores transaction records:

| Column | Type | Description |
|--------|------|-------------|
| id | TEXT (PK) | UUID |
| fromPubKey | TEXT | Sender's public key |
| toPubKey | TEXT | Recipient's public key |
| amount | INTEGER | Amount in cents |
| prevTxHash | TEXT | Previous transaction hash |
| sequenceNumber | INTEGER | Sequence number for chain |
| signature | TEXT | Cryptographic signature |
| status | TEXT | pending/sent/acknowledged/rejected |
| createdAt | INTEGER | Timestamp |
| updatedAt | INTEGER | Timestamp |

## Available Functions

| Function | Description |
|---------|-------------|
| `getLatestEntry()` | Get the latest transaction entry |
| `getNextSequenceNumber()` | Get the next sequence number |
| `getBalance(pubKey)` | Calculate balance (received - sent) |
| `addTransaction(tx)` | Add a new transaction |
| `updateTransactionStatus(id, status)` | Update transaction status |

## Database Initialization

The `SQLiteProvider` component handles database creation:

1. **First launch**: Creates `sqlite-local.db` file
2. **onInit**: Runs `migrateDbIfNeeded()` to create tables
3. **Subsequent launches**: Skips migration if tables exist

The migration uses Drizzle's SQL migrations generated by `drizzle-kit generate`.

## Syncing with Server

When a user logs in and has an existing account on the server, sync their previous transactions:

1. Fetch transactions from server API
2. Use `addTransaction()` to insert each transaction
3. The local ledger maintains correct sequence numbers
